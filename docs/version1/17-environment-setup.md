## 1. Project Initialization (Expo Development Build)

> **Critical:** This app uses native SDKs (DeepAR, RevenueCat, react-native-purchases). It **cannot** run in Expo Go. You must scaffold it as a Development Build project from the start.

```bash
# Create the project

cd groomai

# Add dev client — this is what enables native modules to work
npx expo install expo-dev-client

# Install all dependencies (from 01-project-overview.md package.json)

  expo-apple-authentication expo-media-library expo-linear-gradient expo-blur \
  expo-haptics expo-updates expo-sharing \

  @supabase/supabase-js @react-native-async-storage/async-storage \
  @tanstack/react-query zustand date-fns posthog-react-native \
  @sentry/react-native nativewind

# Build the dev client locally (you need this before running anything)
# iOS (requires a Mac + Xcode):

# Android:
npx expo run:android
```

**You will NOT use `expo start` with Expo Go.** You will use `expo start --dev-client` after the dev client is installed.

## 4. DeepAR Effect Files

> **This is a hidden cost that catches every first-time builder off guard.**

The `assets/deepar-effects/` folder referenced throughout the AR docs must contain real `.deepar` binary 3D effect files — one per hairstyle/beard style. These files are **not generated by code**. You must acquire them.

**Options:**
| Source | Cost | Timeline |
|---|---|---|
| DeepAR Effect Store (deepar.ai/effects) | $20–150/effect | Immediate download |
| Hire a 3D artist (Fiverr, Upwork) | $50–200/effect | 3–7 days/effect |
| DIY in DeepAR Studio (free tool) | Free but complex | 1–3 days/effect |

**Recommended V1 approach:** Start with 5–8 purchased effects to validate the AR feature. Do not try to populate all 50+ styles before launching. Expand the library after you have paying users.

**Folder structure expected by the app:**
```
assets/
  deepar-effects/
    hairstyles/
      mid-fade-01.deepar
      classic-taper.deepar
      french-crop.deepar
      buzz-cut.deepar
      ...
    beards/
      full-beard.deepar
      stubble.deepar
      goatee.deepar
      ...
```

The `deepArEffectId` field in the hairstyles database must exactly match the filename (without `.deepar`).

## 5. Privacy Policy & Legal Requirements (Required for App Store)

This app collects **biometric data** (face photos for skin analysis and AR, hair photos). Both Apple and Google require a privacy policy before submission. Skipping this = app rejection.

### What You Must Disclose

| Data Collected | Where Used | Storage |
|---|---|---|
| Face/skin selfie photos | AI skin analysis | Supabase Storage (private bucket) — user's own folder only |
| Hair loss photos | Monthly hair tracking | Supabase Storage (private bucket) — user's own folder only |
| Face landmarks (AR) | DeepAR on-device processing | **Not stored** — processed locally in real-time |
| Name, email | Account management | Supabase Auth |
| Grooming profile (skin type, hair type, etc.) | Personalized recommendations | Supabase DB |
| Usage analytics | Product improvement | PostHog (anonymized) |
| Error/crash logs | Debugging | Sentry (anonymized) |

### Key Privacy Commitments to Put in Your Policy

1. Face photos are **never shared with third parties**.
2. AR face mesh processing is **on-device only** via DeepAR — no face data leaves the device during AR.
3. Skin analysis photos are sent to OpenAI's API for analysis — disclose this explicitly. OpenAI's API does not train on API data by default (as of 2024 policy).
4. Users can **delete their account and all associated data** at any time (required by GDPR and App Store guidelines).
5. Data deletion must actually work. Implement a "Delete Account" button in Profile → Settings. It must call a Supabase Edge Function (`delete-account`) that performs deletion in this exact order:

```typescript
// supabase/functions/delete-account/index.ts
import { createClient } from '@supabase/supabase-js'

Deno.serve(async (req) => {
  // Only the authenticated user may delete their own account
  const authHeader = req.headers.get('Authorization')
  if (!authHeader) return new Response('Unauthorized', { status: 401 })

  const userClient = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_ANON_KEY')!,
    { global: { headers: { Authorization: authHeader } } }
  )
  const { data: { user }, error: authError } = await userClient.auth.getUser()
  if (authError || !user) return new Response('Unauthorized', { status: 401 })

  const userId = user.id
  const admin = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  // Step 1: Delete all private Storage objects for this user
  // Failure here is logged but does NOT block account deletion —
  // orphaned files are acceptable; orphaned auth credentials are not.
  const storageBuckets = ['skin-analysis', 'hair-loss-photos', 'profile-avatars']
  const storageErrors: string[] = []
  for (const bucket of storageBuckets) {
    const { data: files } = await admin.storage.from(bucket).list(userId)
    if (files && files.length > 0) {
      const paths = files.map(f => `${userId}/${f.name}`)
      const { error } = await admin.storage.from(bucket).remove(paths)
      if (error) storageErrors.push(`${bucket}: ${error.message}`)
    }
  }
  if (storageErrors.length > 0) {
    console.error('Storage deletion partial failure:', storageErrors)
    // Continue — do not block account deletion
  }

  // Step 2: Delete profile row — CASCADE handles all child tables
  // (routines, habit_logs, streaks, user_badges, skin_analysis_logs, etc.)
  await admin.from('profiles').delete().eq('id', userId)

  // Step 3: Delete the auth user (cannot be undone)
  const { error: deleteAuthError } = await admin.auth.admin.deleteUser(userId)
  if (deleteAuthError) {
    return new Response(JSON.stringify({ error: deleteAuthError.message }), { status: 500 })
  }

  return new Response(JSON.stringify({ success: true }), { status: 200 })
})
```

**Failure modes:**
- Storage deletion fails for some files → logged, deletion proceeds (orphaned files can be cleaned up via a scheduled job)
- Profile DELETE fails → return 500, do not delete auth user (safe to retry)
- Auth user DELETE fails after profile is gone → user cannot log in again, but data is already gone; log for manual cleanup

### Minimum Required Legal Pages

- **Privacy Policy** — hosted URL (not in-app only). Use [privacypolicies.com](https://privacypolicies.com) or [iubenda.com](https://iubenda.com) to generate one quickly.
- **Terms of Service** — required for subscriptions.
- **EULA (End User License Agreement)** — required for App Store subscription apps.

Include links to both pages on your App Store listing and in the app (Profile → Settings → Privacy Policy + Terms).

## 8. Error Boundary (Global — Required)

This is not covered in any other file. Add a global error boundary to `app/_layout.tsx` so that unhandled errors show a friendly screen instead of crashing:

```tsx
// components/ErrorBoundary.tsx
import { Component, ReactNode } from 'react'
import { View, Text, Pressable, StyleSheet } from 'react-native'
import * as Updates from 'expo-updates'

interface State { hasError: boolean; error?: Error }

export class ErrorBoundary extends Component<{ children: ReactNode }, State> {
  state: State = { hasError: false }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error }
  }

  componentDidCatch(error: Error) {
    // Log to Sentry
    console.error('Unhandled error:', error)
  }

  async handleRestart() {
    await Updates.reloadAsync()
  }

  render() {
    if (this.state.hasError) {
      return (
        <View style={styles.container}>
          <Text style={styles.title}>Something went wrong</Text>
          <Text style={styles.message}>We logged this automatically. Restart the app to continue.</Text>
          <Pressable style={styles.button} onPress={this.handleRestart}>
            <Text style={styles.buttonText}>Restart App</Text>
          </Pressable>
        </View>
      )
    }
    return this.props.children
  }
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#0A0A0A', justifyContent: 'center', alignItems: 'center', padding: 24 },
  title: { color: '#FFFFFF', fontSize: 22, fontWeight: '700', marginBottom: 12 },
  message: { color: '#A0A0A0', fontSize: 15, textAlign: 'center', marginBottom: 32 },
  button: { backgroundColor: '#C9A84C', paddingHorizontal: 32, paddingVertical: 14, borderRadius: 12 },
  buttonText: { color: '#0A0A0A', fontWeight: '700', fontSize: 16 },
})
```

Wrap the root in `app/_layout.tsx`:
```tsx
return (
  <ErrorBoundary>
    <QueryClientProvider client={queryClient}>
      <Stack screenOptions={{ headerShown: false }}>
        ...
      </Stack>
    </QueryClientProvider>
  </ErrorBoundary>
)
```

## 9. Offline Behavior

Not covered elsewhere. Define this baseline behavior now so it is implemented consistently:

| Scenario | Behavior |
|---|---|
| No internet on launch | Show cached data from React Query (staleTime: 5 min). If no cache, show "You're offline" banner. |
| Completing a routine step offline | Save to local Zustand state + AsyncStorage queue. Sync to Supabase when connection restored. |
| AR try-on offline | v2 only — see `docs/version2/08-ar-tryon.md` (DeepAR effects bundled locally). |
| Barber Translator offline | Works — hairstyle data is seeded in `constants/hairstyles.ts` locally. |
| Skin analysis offline | Show error: "AI analysis requires an internet connection." |
| Product scanner offline | Show error: "Product lookup requires an internet connection." |

Implement a network state hook:
```typescript
// hooks/useNetworkState.ts
import NetInfo from '@react-native-community/netinfo'
import { useEffect, useState } from 'react'

export function useNetworkState() {
  const [isConnected, setIsConnected] = useState(true)

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      setIsConnected(state.isConnected ?? true)
    })
    return unsubscribe
  }, [])

  return { isConnected }
}
```

> `@react-native-community/netinfo` is already listed in `01-project-overview.md`'s `package.json`. Run `npx expo install @react-native-community/netinfo` so Expo pins the correct version for your SDK.
